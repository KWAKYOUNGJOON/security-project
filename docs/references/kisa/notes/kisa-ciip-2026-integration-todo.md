# KISA-CIIP-2026 반영 단계별 TODO

## 문서 목적

이 문서는 `KISA-CIIP-2026`를 우리 프로젝트에 바로 실행 통합하는 것이 아니라, 정적 분석에서 시작해 샘플 출력 수집, 파서 설계, finding 스키마 매핑, `report-automation` 연계, 서버 보고서 템플릿 확장까지 단계적으로 반영하기 위한 작업 계획을 정리한다.

## 전제

- 이번 단계에서는 외부 저장소 원본을 수정하지 않는다.
- `archive/original-sources/**`가 존재할 경우 절대 수정하지 않는다.
- 현재 최우선은 웹 취약점 자동화이며, 서버/웹서버 확장은 그 다음 단계다.
- 이번 분석의 가장 깊은 대상은 `03.웹서버`다.
- 실제 진단 실행은 이번 문서 범위 밖이며, 샘플 출력 수집은 별도 격리 환경에서만 수행해야 한다.
